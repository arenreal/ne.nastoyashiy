Введение в понятие GIT. Работа с GIT.

  • 2020г
  • 03/05 May

Git

  • Что такое git?
  • Для чего он нужен?
  • Как с ним работать?

Я знаю, что сейчас многие начнут орать – мол, таких статей миллион, что он тут нам хочет рассказать. Я согласен ,отчасти, с этим мнением, статей действительно много. Как правило, они либо пытаются охватить чуть больше чем нужно для начала работы(учат присваивать ключи, работать с сетью итд), или же наоборот, недостаточно подробные (отсутствует связь с gitHUB). Поэтому, собственно, данная статья здесь и появилась.

Итак, после небольшого вступления, давайте попробуем разобраться в вопросах озвученных выше и научиться пользоваться гитом. Для начала разберемся, что же это такое? Git – это распределенная система контроля версиями. Если простым языком – программа, позволяющая вам контролировать, а именно применять и откатывать, изменения, которые вы производили в проекте. С помощью гита вы можете создавать ветки на своем проекте и производить на них ряд конкретных изменений, которые потом войдут в проект или, не понадобившись, не навредят, также сможете легко переключаться между ранними и поздними вариантами, вносить и отслеживать изменения.

Система позволяет работать большой команде над одним проектом, да и даже, работая вдвоем, будет намного проще производить изменения и решать разногласия и конфликты. Систем контроля версий много и все они работают по принципу компьютерной игры, где вы можете вернуться к месту сохранения, если что-то пошло не так. GIT является наиболее популярной из таких систем

Git был разработан Линусом Торвальдсом, отсюда и название GIT (пер.с .англ.как мерзавец) , свойственное данному разработчику, так как он говорил, что называет все программы в честь себя, а себя считал эгоистичным ублюдком…”Бла бла бла. Давай уже код, зануда”… Ок! Поехали.

Для начала работы нам нужно будет установить GIT на локальную машину. Это можно сделать на официальном сайте программы (https://git-scm.com/downloads).

Выбираете версию подходящую для вашего Windows: 32 или 64 разрядную, или же для Mac или Linux. Ставить нужно только полную версию, а не Portable. Установив программу можно начинать знакомство.

Отличие GIT от других программ в том, что у него отсутствует графическая версия. Поэтому, работа с Git ведётся через командную строку. В разных операционных системах - свои программы для взаимодействия с Git, но начинающим пользователям, я бы рекомендовал освоить классический подход, а именно работу через терминал. После установки GIT, вам будет доступен его собственный терминал, через который очень удобно работать, также можете использовать стандартную командную строку Windows. После освоения данных методов – можно выбрать удобную оболочку с графическим интерфейсом, подходящую именно вам. Таких, великое множество. Я сам пользуюсь обычной командной строкой или терминалом GIT, работа в таких средах лучше помогает понять сам принцип , GIT..

Для начала давайте разберемся с основными операторами, клавишами и их сочетаниями командной строки. Все разбирать здесь не будем, для этого есть отдельная статья (Справочник команд.), мы же, рассмотрим только те, что нам понадобятся в рамках данной статьи.

Клавиши(работают во всех ОС).

  • ↑,↓ — стрелки вверх и вниз в командной строке позволяют смотреть предыдущие введенные команды, чтобы 2 раза не набирать.
  • В Windows при закрытии окошка консоли вся история теряется, в других ОС — сохраняется
  • [tab] — автодополняет имена файлов и команды. Если в текущем каталоге есть файл file.txt, ты наберешь fi и нажмешь [tab], то он автодополнит имя файла (или предложит варианты, если таких файлов несколько).
  • [ctrl+c] — прерывает выполнение команды, например, если она долго тупит.
  • Важно: клавиша Esc, как и многие привычные клавиши, не работает в командной строке. Клавиши Ctrl + C, V, X обычно не работают как «копировать/вырезать/вставить» (иногда для вставки можно использовать Shift + Ins). Клавиша Ctrl + Q не работает как «выход».

Команды.

В командной строке, результат выполнения команд часто зависит от того, в каком каталоге, и на каком диске, ты находишься. Также, когда ты указываешь в команде относительное имя файла (без полного пути и имени диска), оно ищется в текущем каталоге. Какой каталог текущий, можно узнать командой

pwd
(print working directory), она работает не во всех ОС, и не во всех оболочках. Как правило, если данная команда не применяется на текущей версии ОС, внизу, рядом с курсором, находится подсказка – указывающая на ваше текущее месторасположение.

Чтобы перейти в другой каталог, используют команду

cd
  • cd / — перейти в корневой каталог.
  • cdfiles — переходит в каталог files внутри текущего.
  • cd .. — переходит в родительский каталог.
  • cd ../.. — переходит на 2 уровня выше.
Очистка экрана консоли в windows делается с помощбю команды (clear screen)
cls
список файлов в каталоге ожно узнать с помощью команды
ls
  • ls — показать список файлов.
  • ls -l — показать файлы с подробностями.
На этом, в принципе, можно закончить, для первоначальной работы нам этого хватит.

Давайте теперь подробнее разберемся в терминах GIt и его структуре в целом. Гит представляет собой дерево с ветками. Основной ствол дерева является главной веткой, называется она master. Эта корневая ветка проекта, к которой примыкают все остальные. Остальных веток может быть любое количество. Такое дерево позволяет нам, создав ветку на проекте, заниматься разработкой отдельной части проекта, не вмешиваясь в исходный код, проверять и тестить изменения, и уже потом, вливать их в ветку master. В этом основная суть GIT.

С помощью терминала проверим, что Git действительно установлен.Откройте терминал и введите команду

git
Эта проверка подходит для всех операционных систем. На экране должно появиться краткое описание основных команд GIT. Их-то как раз мы и будем разбирать сегодня.

Настройка Git.

После того как Git появился на компьютере, нужно ввести свои данные, а именно - имя и адрес электронной почты. Ваши действия в Git будут содержать эту информацию. Для этого в терминале используйте следующую команду, чтобы добавить своё имя:

git config --global user.name "ваше имя"
Для добавления почтового адреса вводите:
git config --globaluser.email “ваш адрес”

Обратите внимание, что в командах, указанных выше, есть опция --global. Это значит, что такие данные будут сохранены для всех ваших действий в Git и вводить их больше не надо. Если вы хотите менять эту информацию для разных проектов, то в директории проекта вводите эти же команды, только без опции --global.

Теперь можно считать что GIT настроен, но без GITHub он малоинтересен.

GitHub.

GitHub— веб-сервис, который основан на системе Git. Это такая социальная сеть для разработчиков, которая помогает удобно вести коллективную разработку IT-проектов. Здесь можно публиковать и редактировать свой код, комментировать чужие наработки, следить за новостями других пользователей, ну и собственно пользоваться самим GIT. Потому мы пройдем там регистрацию и заведем там свой профиль. Помимо gitHub есть и другие альтернативные веб- сервисы типа GitLab, но в данной статье мы рассмотрим совместную работу именно с GitHub.

Перейдем на сайт GitHub.

Здесь все просто – вводим свои данные, подтверждаем почту и бла бла бла, регистрацию, я думаю, все смогут пройти, в противном случае, возможно, стоит задать себе вопрос – А зачем я в это лезу?

После создания профиля мы увидим главную страницу, что нам может здесь быть интересно? Да по факту две вещи – создание нового репозитория и список уже имеющихся. Если вы только что прошли регистрацию, то у вас список будет пустым. Давайте сразу же создадим новый репозиторий, он понадобиться нам ниже. Не будем пока глубоко вдаваться что такое репозиторий – это станет ясно чуть ниже, пока просто ассоциируйте это с созданием нового проекта, хотя это не совсем точно. Для создания нового репозитория нажмем на зелененькую кнопку NEW

  • Введем название проекта, пускай у нас это будет project01
  • Выберем тип репозитория, он может быть как публичным (доступным для всех), так и приватным (доступным вам одному). По умолчанию, он встает на публичный. Мы не будем глубоко вдаваться в эти понятия сейчас, просто выберем публичный.
  • Ставим галочку о создании файла Readme. В общем и целом, это хорошая практика, создавать такой файл и описывать в нем подробности проекта. В нашем же случае, мы создадим его для того, чтобы не создавать пустого репозитория.
  • Ну и собственно подтверждение.

Отлично мы создали новый репозиторий с нашим проектом на GITHUB. Позже мы еще вернемся к этому вопросу. Пока оставим эту страницу в браузере открытой и снова перейдем в консоль.
Не устали? Щас все будет уже немного сложнее, но как по мне – интереснее. GIT – это волшебник на вашем компьютере, и вы в этом убедитесь.

Давайте создадим папку с нашим проектом в системе. Я буду создавать прямо на рабочем столе чтобы не мучиться. Создав ее, перейдем туда через терминал с помощью команды cd. Посмотрим где мы находимся. Я нахожусь в папке, на один уровень выше рабочего стола, поэтому я поднимусь до рабочего стола и провалюсь в папку с проектом.

Отлично! Создадим в этой папке репозиторий. Для этого нам придется использовать команду:

git init
.

Система даст нам ответ, о том, что был создан файл репозитория .git, это скрытый файл, и вы его не увидите, если только у вас не настроен показ скрытых файлов, это довольно легко можно сделать с помощью настроек проводника.

Что мы только что сделали ?– мы создали репозиторий с основной веткой внутри master. Теперь GIT будет отслеживать все изменения и запоминать их.Изменения ,на данный момент, будут касаться только нашей ветки master, так как она, пока что, единственная, мы коснемся этого вопроса чуть подробнее позже, пока достаточно знать лишь то, что в нашей папке проекта создан репозиторий.

Введем главную команду

git status

Система даст нам ответ о том, что мы находимся на ветке master, новых commit нет, и никаких изменений на ней - не замечено, попробуем это исправить. Для этого создадим в папке нашего проекта файл. Например, index.html, и заполним его стандартной формой с одним маленьким параграфом “Привет”.

Создавать можно любые файлы, картинку например, неважно. Я создаю стандартную разметку для страницы, для того, чтобы пойзже показать как можно разместить свой проект на сервере gitHUB и получить на него прямую ссылку, которой вы сможете делиться c друзьями. И как вы, наверное, понимаете – картинка для этого не лучший вариант, потому и использую заготовку страницы. Все изменения в коде я делаю в редакторе Sublime Text 3.
